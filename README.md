# MCHS Project API

Это backend для мобильного приложения, которое помогает обучать и тестировать сотрудников МЧС. Основная фишка проекта — можно загрузить PDF или Word документ с вопросами, и система автоматически создаст из него тест. Не нужно руками вбивать каждый вопрос и вариант ответа.

## Что использовал

Проект написан на **.NET 10** с **ASP.NET Core**. Для базы данных выбрал **PostgreSQL**, а вместо тяжелого Entity Framework использую **Dapper** — он быстрее и даёт больше контроля над SQL запросами.

Для безопасности реализована JWT аутентификация с refresh токенами. Пароли хешируются через **BCrypt** с work factor 12, так что даже если база утечёт, пароли не восстановить.

Парсинг документов делаю через **iText7** для PDF и **OpenXml** для DOCX файлов. Swagger подключен для удобного тестирования API.

### Полный список пакетов:

- ASP.NET Core 10.0
- PostgreSQL 16+ с Npgsql 10.0.1
- Dapper 2.1.66
- JWT Bearer Authentication 10.0.1
- BCrypt.Net-Next 4.0.3
- iText7 9.0.0
- DocumentFormat.OpenXml 3.2.0
- Swashbuckle.AspNetCore 7.2.0

## Что умеет

**Аутентификация** — регистрация, вход, выход, смена пароля. Токены живут час, но есть refresh token на 30 дней, чтобы не логиниться постоянно.

**Лекции** — создание учебных материалов с привязкой видео и документов.

**Тесты** — можно создать вручную или загрузить документ. Система сама распознает вопросы, варианты ответов и правильные ответы.

**Результаты** — сохраняется история прохождения тестов, баллы, время.

**Отчёты** — JSON отчёты по прогрессу обучения.

## Как запустить

Нужен .NET SDK 10.0 и PostgreSQL 16+.

1. Склонируй репозиторий
2. Создай базу данных и выполни SQL скрипт из `database.sql`
3. Настрой `appsettings.json`:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=mchs_db;Username=postgres;Password=твой_пароль"
  },
  "Jwt": {
    "Secret": "секретный-ключ-минимум-32-символа-для-jwt-токенов",
    "Issuer": "MCHSProject",
    "Audience": "MCHSMobileApp",
    "AccessTokenExpirationMinutes": 60,
    "RefreshTokenExpirationDays": 30
  }
}
```

4. Запусти:
```bash
dotnet restore
dotnet run
```

API будет на `https://localhost:7265`, Swagger на `https://localhost:7265/swagger`

## API

### Аутентификация

Для защищённых эндпоинтов нужен токен в заголовке:
```
Authorization: Bearer твой_токен
```

**Регистрация** — `POST /api/Auth/register`
```json
{
  "username": "ivanov",
  "password": "SecurePass123!",
  "roleId": 2
}
```
roleId: 1 — админ, 2 — преподаватель, 3 — студент

**Вход** — `POST /api/Auth/login`
```json
{
  "username": "ivanov",
  "password": "SecurePass123!"
}
```
Вернёт accessToken и refreshToken.

**Обновление токена** — `POST /api/Auth/refresh-token`
```json
{
  "refreshToken": "твой_refresh_token"
}
```

**Смена пароля** — `POST /api/Auth/change-password` (нужен токен)
```json
{
  "oldPassword": "СтарыйПароль",
  "newPassword": "НовыйПароль123!"
}
```

**Выход** — `POST /api/Auth/logout` (нужен токен)

### Лекции

- `GET /api/Lectures` — все лекции
- `GET /api/Lectures/{id}` — одна лекция
- `POST /api/Lectures` — создать (нужен токен)
- `PUT /api/Lectures/{id}` — обновить
- `DELETE /api/Lectures/{id}` — удалить

Пример создания:
```json
{
  "title": "Основы пожарной безопасности",
  "textContent": "Текст лекции...",
  "pathId": 1
}
```

### Тесты

- `GET /api/Tests` — все тесты
- `GET /api/Tests/{id}` — один тест
- `GET /api/Tests/by-lecture/{lectureId}` — тесты по лекции
- `POST /api/Tests` — создать
- `PUT /api/Tests/{id}` — обновить
- `DELETE /api/Tests/{id}` — удалить

### Вопросы и ответы

- `GET /api/Questions/by-test/{testId}` — вопросы теста
- `POST /api/Questions` — добавить вопрос
- `GET /api/Answers/by-question/{questionId}` — ответы на вопрос
- `POST /api/Answers` — добавить ответ

### Результаты тестов

- `GET /api/TestResults/by-user/{userId}` — результаты пользователя
- `GET /api/TestResults/by-test/{testId}` — результаты по тесту
- `POST /api/TestResults` — сохранить результат

## 🔥 Создание теста из документа

Самая полезная фича. Загружаешь PDF или DOCX с вопросами, и система сама всё распарсит.

**Эндпоинт:** `POST /api/Documents/create-test`

Отправляешь form-data:
- `lectureId` — к какой лекции привязать
- `testTitle` — название теста
- `createdBy` — ID автора
- `documentFile` — сам файл

Пример через curl:
```bash
curl -X POST "https://localhost:7265/api/Documents/create-test" \
  -H "Authorization: Bearer твой_токен" \
  -F "lectureId=1" \
  -F "testTitle=Тест по пожарной безопасности" \
  -F "createdBy=2" \
  -F "documentFile=@test.pdf"
```

### Как правильно оформить документ

Чтобы парсер понял структуру, нужно соблюдать формат:

```
Вопрос 1: Что делать при обнаружении пожара?
а) Вызвать пожарную охрану по номеру 101 [правильно]
б) Попытаться потушить огонь самостоятельно
в) Убежать и никому не сообщать
г) Открыть окна для проветривания

Вопрос 2: Какой огнетушитель для электроприборов?
а) Водный
б) Пенный
в) Углекислотный [правильно]
г) Порошковый

Вопрос 3: Как покидать задымлённое помещение?
а) Бежать в полный рост
б) Передвигаться пригнувшись или ползком [правильно]
в) Ждать помощи на месте
г) Открыть все двери
```

**Вопросы** начинаются с `Вопрос N:` или `Question N:` — номер обязателен.

**Ответы** — буква + скобка: `а)`, `б)`, `в)` или `a)`, `b)`, `c)`. Можно цифры: `1)`, `2)`, `3)`.

**Правильный ответ** помечается одним из способов:
- `[правильно]` или `[correct]` в конце
- Двойные звёздочки `**ответ**`
- Галочка `✓` или стрелка `➔`

Если формат соблюдён, система создаст тест со всеми вопросами и ответами автоматически.

## База данных

Схема простая:

- **users** — пользователи с хешированными паролями
- **roles** — роли (админ, преподаватель, студент)
- **lectures** — учебные материалы
- **paths** — пути к медиа файлам
- **tests** — тесты, привязанные к лекциям
- **questions** — вопросы тестов
- **answers** — варианты ответов с пометкой правильного
- **test_results** — результаты прохождения
- **user_answers** — что отвечал пользователь
- **reports** — отчёты в JSON формате

SQL скрипт для создания таблиц лежит в корне проекта.

## Безопасность

- Пароли хешируются BCrypt с work factor 12
- JWT токены с коротким сроком жизни (1 час)
- Refresh токены для продления сессии
- Все SQL запросы параметризованы (защита от инъекций)
- CORS настроен для мобильных приложений
- Валидация всех входящих данных
